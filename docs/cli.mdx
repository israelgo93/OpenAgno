La ruta de operación documentada para OpenAgno es:

```bash
openagno <comando>
```

`python -m management.cli` sigue existiendo como compatibilidad interna, pero está deprecado como punto de entrada para usuarios.

## Comandos principales

| Comando | Uso |
|---------|-----|
| `openagno init --template <id>` | Crear workspace desde template |
| `openagno templates list` | Listar templates disponibles |
| `openagno templates show <id>` | Ver metadata de un template |
| `openagno validate` | Validar `workspace/` |
| `openagno start --foreground` | Ejecutar el runtime en foreground |
| `openagno start` | Arrancar supervisor local |
| `openagno status` | Estado del runtime |
| `openagno logs --follow` | Seguir `gateway.log` |
| `openagno stop` | Detener supervisor |
| `openagno restart` | Reiniciar supervisor |

## Comandos de workspace

| Comando | Uso |
|---------|-----|
| `openagno create agent "<nombre>"` | Crear sub-agente |
| `openagno add whatsapp --mode <modo>` | Configurar WhatsApp |
| `openagno add slack` | Agregar Slack |
| `openagno add telegram` | Agregar Telegram |
| `openagno add agui` | Agregar AG-UI |
| `openagno add a2a` | Habilitar A2A |
| `openagno add tool <tool>` | Activar o desactivar tool opcional |
| `openagno deploy docker` | Levantar stack local con Docker Compose |
| `openagno deploy local` | Alias del supervisor local |

## Salida esperada

La CLI usa un patrón consistente: **header + pasos + next step**.

### `openagno init --template personal_assistant`

```text
Initializing OpenAgno...
    Copying template `personal_assistant` into `/path/to/workspace`
    ✔ Workspace initialized from packaged template.
    → Next: Fill in `.env`, then run `openagno validate`.
```

### `openagno add slack`

```text
Updating workspace...
    Adding Slack channel
    ✔ Slack channel added.
    → Next: Set `SLACK_TOKEN` and `SLACK_SIGNING_SECRET` in `.env`.
```

### `openagno start`

```text
Starting OpenAgno...
    Launching the local supervisor in background.
    ✔ Supervisor started in background with PID 12345.
    → Next: Run `openagno status` or `openagno logs --follow`.
```

## Ejemplo completo

```bash
openagno init --template customer_support
openagno add slack
openagno add agui
openagno add tool tavily
openagno validate
openagno start --foreground
```

## Alcance actual

- `openagno deploy aws` todavía no provisiona infraestructura.
- La CLI trabaja sobre `OPENAGNO_ROOT/workspace`.
- Si usas supervisor, los logs quedan en `gateway.log`.
- `openagno init` sin template sigue lanzando el wizard interactivo legacy como bridge interno.
