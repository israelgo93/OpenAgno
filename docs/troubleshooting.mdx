title: Troubleshooting
description: Problemas comunes al arrancar u operar OpenAgno y como diagnosticarlos sin perder tiempo.
---

## El gateway no arranca

Verifica primero:

```bash
openagno validate
openagno start --foreground
```

Si usas supervisor:

```bash
openagno status
openagno logs --follow
```

## `Workspace no encontrado`

La CLI empaquetada espera `workspace/` dentro de `OPENAGNO_ROOT` o del directorio actual.

Acciones:

- ejecuta comandos desde la raiz del proyecto,
- o define `OPENAGNO_ROOT`,
- o inicializa el workspace con `openagno init --template ...`.

## Knowledge deshabilitada

Si ves que la knowledge devuelve vacio o no se inicializa:

- revisa si `database.type` es `sqlite`,
- confirma que PgVector este disponible,
- valida `OPENAI_API_KEY` si usas el embedder por defecto.

## WhatsApp Cloud API responde `Invalid signature`

Revisa:

- `WHATSAPP_APP_SECRET`
- que Meta este firmando la request correcta
- que no estes haciendo pruebas locales sin firma real

Para pruebas locales controladas:

```bash
WHATSAPP_SKIP_SIGNATURE_VALIDATION=true
```

## Mensajes repetidos en WhatsApp

OpenAgno deduplica por `message_id`, pero solo despues de validar firma. Si ves replays:

- confirma que el proveedor externo realmente reenvia el mismo `message_id`,
- revisa logs de `Webhook duplicado ignorado`,
- prueba con un payload mixto para confirmar que solo se filtran los repetidos.

## Warnings de dependencias HTTP/WebSocket

La combinacion validada en esta fase usa:

- `chardet>=5.2,<6`
- `wsproto>=1.3.0`

Si reaparecen warnings, revisa que el entorno virtual no haya quedado con versiones fuera de `requirements.lock`.

## La CLI no encuentra logs

`openagno logs` lee `gateway.log`. Ese archivo existe cuando arrancas con:

- `openagno start`
- `python service_manager.py start`

Si arrancas en foreground, los logs salen a stdout y no necesariamente a `gateway.log`.
