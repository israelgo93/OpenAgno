OpenAgno es una plataforma sobre **Agno** para construir y operar agentes mediante un **workspace declarativo**. Combina una CLI empaquetada, un runtime AgentOS/FastAPI, templates reutilizables, canales, knowledge con PgVector y soporte MCP.

## Lo que existe hoy

- CLI `openagno` publicada en PyPI
- workspace declarativo en YAML y Markdown
- runtime AgentOS con rutas operativas, multi-tenant y knowledge
- templates empaquetados
- canales: WhatsApp Cloud API (single-tenant y multi-tenant con webhook por tenant), WhatsApp QR via bridge Baileys, Slack, Telegram
- soporte opcional para AG-UI y A2A
- PgVector sobre PostgreSQL o Supabase
- aprovisionamiento multi-tenant con workspaces y filtros de knowledge por tenant, cache LRU (`TenantLoader`)
- credenciales BYOK por tenant con fallback seguro a variables de entorno del servidor
- audio por Whisper cuando el modelo del tenant no soporta audio nativo
- cliente y servidor MCP
- exportaciones MCP para IDEs y `AGENTS.md` + skills agnosticas para IDEs con capacidades de agente
- documentacion Mintlify bilingue (ingles en la raiz, espanol bajo `docs/es/`)
- licencia Apache 2.0, guia de contribuidor completa, politica de seguridad y templates de issue/PR

## Flujo recomendado

1. Instala el proyecto.
2. Crea un workspace desde un template.
3. Valida la configuracion.
4. Arranca el runtime con `openagno`.
5. Conecta canales, IDEs o clientes externos.

La documentacion publica tambien expone un endpoint MCP en `https://docs.openagno.com/mcp` y archivos AI-friendly en `https://docs.openagno.com/llms.txt`.
