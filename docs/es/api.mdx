---
title: API HTTP
description: Endpoints del runtime OpenAgno, incluyendo provisioning multi-tenant, knowledge, admin y rutas nativas de AgentOS.
---

Base URL tipica: `http://localhost:8000` (ajusta host y puerto segun tu despliegue).

## Rutas custom de OpenAgno

| Metodo | Ruta | Descripcion |
|--------|------|-------------|
| `GET` | `/admin/health` | Estado del runtime, canales, agentes, modelo y scheduler |
| `POST` | `/admin/reload` | Solicita al supervisor recargar el runtime |
| `POST` | `/admin/fallback/activate` | Activa manualmente el modelo de fallback |
| `POST` | `/admin/fallback/restore` | Restaura el modelo principal |

Estas rutas tienen rate limit. Si las expones en produccion, dejalas detras de red privada o de un reverse proxy.

## Rutas de knowledge

| Metodo | Ruta | Descripcion |
|--------|------|-------------|
| `POST` | `/knowledge/upload` | Sube un documento |
| `POST` | `/knowledge/ingest-urls` | Ingiere URLs |
| `GET` | `/knowledge/list` | Lista documentos indexados |
| `DELETE` | `/knowledge/{doc_name}` | Elimina un documento |
| `POST` | `/knowledge/search` | Ejecuta busqueda semantica |

Si envias `X-Tenant-ID`, estas rutas pasan a ser tenant-aware y usan aislamiento nativo de Agno.

Si defines `OPENAGNO_API_KEY`, estas rutas exigen el header `X-API-Key`.

## Rutas de provisioning multi-tenant

| Metodo | Ruta | Descripcion |
|--------|------|-------------|
| `GET` | `/tenants` | Lista tenants |
| `POST` | `/tenants` | Crea un tenant y provisiona su workspace |
| `GET` | `/tenants/{tenant_id}` | Lee metadata del tenant |
| `PATCH` | `/tenants/{tenant_id}` | Actualiza plan, limites, slug o config del workspace |
| `DELETE` | `/tenants/{tenant_id}` | Desactiva el tenant |
| `GET` | `/tenants/{tenant_id}/workspace` | Lee la configuracion del workspace |
| `PUT` | `/tenants/{tenant_id}/workspace` | Actualiza la configuracion del workspace |
| `GET` | `/tenants/{tenant_id}/workspace/inventory` | Lee agente principal, sub-agentes, teams, modelo y estado de WorkspaceTools cargados |
| `POST` | `/tenants/{tenant_id}/workspace/sub-agents` | Crea o actualiza un sub-agente del tenant mediante WorkspaceTools |
| `POST` | `/tenants/{tenant_id}/workspace/sub-agents/{agent_id}/disable` | Deshabilita un sub-agente del tenant |
| `DELETE` | `/tenants/{tenant_id}/workspace/sub-agents/{agent_id}` | Elimina un sub-agente del tenant |
| `POST` | `/tenants/{tenant_id}/workspace/teams` | Crea o actualiza un team del tenant mediante WorkspaceTools |
| `POST` | `/tenants/{tenant_id}/workspace/teams/{team_id}/disable` | Deshabilita un team del tenant |
| `DELETE` | `/tenants/{tenant_id}/workspace/teams/{team_id}` | Elimina un team del tenant |
| `POST` | `/tenants/{tenant_id}/reload` | Invalida y recarga la cache runtime del tenant |
| `POST` | `/tenants/{tenant_id}/agents/{agent_id}/runs` | Ejecuta un agente con `knowledge_filters` tenant-aware |

Las rutas de mutacion de workspace estan pensadas para control planes externos que administran tenants via HTTP. Mantene `WorkspaceTools` habilitado para tenants que pueden auto-configurar sub-agentes y teams, y ejecuta reload despues de las mutaciones antes de esperar cambios en el inventario cargado.

|--------|------|-------------|

## WhatsApp Cloud API

| Metodo | Ruta | Descripcion |
|--------|------|-------------|
| `GET` | `/whatsapp/webhook` | Verificacion de webhook con Meta (un solo tenant, env vars) |
| `POST` | `/whatsapp/webhook` | Eventos de WhatsApp Cloud API (un solo tenant) |
| `GET` | `/whatsapp-cloud/{tenant_id}/webhook` | Verificacion multi-tenant; `tenant_id` es el UUID de `tenants.id` en Cloud |
| `POST` | `/whatsapp-cloud/{tenant_id}/webhook` | Eventos multi-tenant con credenciales cifradas por tenant (requiere `CHANNEL_SECRETS_KEY`) |

## Rutas nativas de AgentOS

El runtime tambien expone rutas nativas de AgentOS. Los grupos operativos mas importantes son:

| Grupo | Rutas principales |
|-------|-------------------|
| Agentes | `/agents`, `/agents/{agent_id}/runs` |
| Teams | `/teams`, `/teams/{team_id}/runs` |
| Sesiones | `/sessions`, `/sessions/{session_id}`, `/sessions/{session_id}/runs` |
| Memorias | `/memories`, `/memory_topics`, `/optimize-memories` |
| Schedules | `/schedules`, `/schedules/{schedule_id}`, `/schedules/{schedule_id}/trigger` |
| Registry / config | `/registry`, `/config`, `/components`, `/models` |
| Observabilidad | `/health`, `/metrics`, `/traces`, `/eval-runs` |

## Ejemplo

Chequeo de salud:

```bash
curl http://127.0.0.1:8000/admin/health
```

Busqueda custom sobre knowledge:

```bash
curl -X POST http://127.0.0.1:8000/knowledge/search \
  -H 'Content-Type: application/json' \
  -d '{"query":"OpenAgno","max_results":3}'
```

Ejecucion tenant-aware:

```bash
curl -X POST http://127.0.0.1:8000/tenants/acme/agents/agnobot-main/runs \
  -H 'Content-Type: application/json' \
  -H 'X-API-Key: change-me' \
  -d '{"message":"Resume mis documentos","user_id":"demo"}'
```
