OpenAgno esta disenado para trabajar con cualquier IDE con capacidades de agente sin casarse con una convencion especifica del vendor. Hay tres capas combinables:

1. **`AGENTS.md`** en la raiz del repo &mdash; el archivo estandar que cualquier agente AI deberia leer primero. Describe el alcance del repo, el contrato del runtime, comandos de setup y reglas del proyecto.
2. **Skills en `.agents/skills/`** &mdash; playbooks enfocados para tareas frecuentes. Hoy el repo trae `openagno`, `openagno-channels` y `mintlify`.
3. **Integracion MCP** &mdash; le suma al agente documentacion autoritativa y al dia de OpenAgno y sus dependencias.

Juntas permiten que cualquier IDE que entienda MCP y `AGENTS.md` sea productivo en el repo sin necesidad de prompt engineering desde cero.

## Endpoints publicos MCP

- Servidor MCP: `https://docs.openagno.com/mcp`
- `llms.txt`: `https://docs.openagno.com/llms.txt`
- `llms-full.txt`: `https://docs.openagno.com/llms-full.txt`

## Convencion `AGENTS.md`

El repositorio sigue el estandar [`agents.md`](https://agents.md), adoptado de facto por la mayoria de asistentes AI modernos. Si tu IDE busca `AGENTS.md`, `CLAUDE.md`, `.cursorrules` o `rules.md`, leera el archivo directamente o podras apuntarlo a el. El contenido es agnostico al IDE a proposito.

Ubicacion:

- `AGENTS.md` en la raiz de `OpenAgno`

Contenido:

- Alcance del repositorio y limites de su contrato HTTP publico
- Baseline actual (version, piso de Agno, Python, Node)
- Prerrequisitos para correr el proyecto
- Comandos de setup
- Mapa del repositorio
- Comandos de validacion y expectativas por tipo de cambio
- Contrato HTTP publico que consumen sistemas externos
- Reglas de seguridad
- Checklist de workflow para agentes

## Skills en `.agents/skills/`

Cada skill es un archivo Markdown autocontenido con frontmatter YAML que describe cuando aplicarlo. Los IDEs que entienden la convencion de skills (Claude Code, Cursor, etc.) los exponen automaticamente cuando coinciden con la tarea del usuario. En IDEs que no la entienden, los archivos siguen sirviendo como documentacion buscable.

Este repo trae:

- `.agents/skills/openagno/SKILL.md` &mdash; el playbook general del runtime y del CLI. Cubre instalacion, comandos comunes, paths clave, rutas del runtime y tareas tipicas (anadir sub-agente, anadir canal, BYOK, debuggear una ejecucion).
- `.agents/skills/openagno-channels/SKILL.md` &mdash; WhatsApp Cloud API (single-tenant y multi-tenant), WhatsApp QR (bridge Baileys), Slack, Telegram, AG-UI, A2A. Incluye setup de Meta Developer Console y tips de debugging.
- `.agents/skills/mintlify/SKILL.md` &mdash; mejores practicas para escribir y mantener la documentacion.


## Setup por IDE

Plantillas listas para copiar en el repositorio:

- `ide-configs/cursor-mcp.json`
- `ide-configs/vscode-mcp.json`
- `ide-configs/windsurf-mcp.json`
- `ide-configs/claude-code-setup.sh`

Ruta recomendada por herramienta:

- **Claude Code**: `bash ide-configs/claude-code-setup.sh`. Claude Code lee `AGENTS.md` (o el legacy `CLAUDE.md`) automaticamente y expone las skills como slash commands.
- **Cursor**: copia `ide-configs/cursor-mcp.json` a `.cursor/mcp.json`. Cursor tambien lee `AGENTS.md` como contexto del proyecto.
- **VS Code** (GitHub Copilot): copia `ide-configs/vscode-mcp.json` a `.vscode/mcp.json`. Copilot Workspace lee `AGENTS.md` cuando existe.
- **Windsurf**: integra `ide-configs/windsurf-mcp.json` en `~/.codeium/windsurf/mcp_config.json`.
- **Aider, Zed, OpenHands o cualquier otro IDE de agente**: apuntalos a `https://docs.openagno.com/mcp` como servidor MCP remoto y asegurate de que el `AGENTS.md` del repo este en la ventana de contexto del agente. Nada mas hace falta.

## Para operadores y usuarios no tecnicos

Si tu herramienta tiene interfaz para MCP, agrega un servidor remoto y pega:

```text
https://docs.openagno.com/mcp
```

Si tu herramienta solo soporta ingestar URLs, usa `llms.txt` o `llms-full.txt`.

## Opcion local

Si ejecutas OpenAgno localmente y mantienes `agentos.enable_mcp_server: true`, el runtime tambien expone su propio MCP. Eso sirve cuando quieres que tu IDE consulte tus agentes, tus tools y tu workspace, no solo la documentacion publica.
