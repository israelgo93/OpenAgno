```text
OpenAgno/
  openagno/            # CLI, helpers y templates empaquetados
  workspace/           # Configuración declarativa del agente
  gateway.py           # Runtime AgentOS/FastAPI
  loader.py            # Construcción de agentes, DB, knowledge y MCP
  routes/              # Endpoints REST custom
  tools/               # Tools locales del runtime
  tests/               # Suite de regresión
  docs/                # Sitio Mintlify
  bridges/whatsapp-qr/ # Bridge opcional para QR
  deploy/              # Assets de despliegue y service units
  docker-compose.yml   # Stack local
  pyproject.toml       # Paquete e instalación
```

## Qué toca cada área

- `openagno/`: superficie pública de la CLI y templates distribuibles
- `workspace/`: comportamiento declarativo del agente
- `gateway.py`: runtime, interfaces, rate limiting, deduplicación y admin
- `loader.py`: traduce el workspace a objetos Agno
- `routes/`: endpoints extra sobre AgentOS
- `tools/`: tools propios del proyecto
- `tests/`: cobertura mínima del runtime y la CLI
- `docs/`: documentación pública

Para entender el flujo entre estas piezas, revisa [Arquitectura](/architecture).
