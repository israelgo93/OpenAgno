## Que toca cada area

- `openagno/`: CLI empaquetada y templates distribuidos con el proyecto
- `gateway.py`: runtime principal, interfaces, dedup, rate limit, admin
- `loader.py`: traduccion del workspace a objetos Agno
- `workspace/`: configuracion declarativa del agente
- `routes/`: rutas REST custom agregadas sobre AgentOS
- `management/`: utilidades legacy e interactivas
- `tests/`: regresiones del runtime, CLI y helpers
- `docs/`: sitio Mintlify

Para la relacion entre componentes, revisa [Arquitectura](/architecture).
