OpenAgno hoy se instala **desde el repositorio**. La publicación a PyPI sigue en preparación para maintainers, así que la ruta pública recomendada continúa siendo `pip install -e .`.

## Requisitos

- Python 3.10 o superior. Python 3.11+ es la opción recomendada.
- PostgreSQL con `pgvector` si quieres knowledge vectorial.
- Node.js 18+ solo si usarás WhatsApp QR o validar Mintlify localmente.

## Ruta rápida

```bash
git clone https://github.com/israelgo93/OpenAgno.git
cd OpenAgno
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
openagno init --template personal_assistant
openagno validate
openagno start --foreground
```

## Ruta para contributors o maintainers

Si además vas a correr tests, preparar builds o habilitar AG-UI/A2A:

```bash
pip install -e '.[dev,protocols]'
```

Para preparar un build local del paquete:

```bash
python -m build
```

## Explora templates disponibles

```bash
openagno templates list
openagno templates show personal_assistant
```

## Completa `.env`

Variables comunes:

- `GOOGLE_API_KEY`, `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`
- `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`
- `WHATSAPP_ACCESS_TOKEN`, `WHATSAPP_PHONE_NUMBER_ID`, `WHATSAPP_VERIFY_TOKEN`
- `SLACK_TOKEN`, `SLACK_SIGNING_SECRET`
- `TELEGRAM_TOKEN`
- `OPENAGNO_API_KEY` si quieres proteger rutas custom

## Arranque recomendado

Foreground:

```bash
openagno start --foreground
```

Supervisor local:

```bash
openagno start
openagno status
openagno logs --follow
```

## Verifica salud

```bash
curl http://127.0.0.1:8000/admin/health
```

## Notas prácticas

- Si eliges `database.type: sqlite`, OpenAgno corre, pero sin knowledge vectorial.
- Slack y Telegram forman parte de la instalación base del proyecto.
- `agui` y `a2a` requieren `pip install -e '.[protocols]'`.
- Si vas a usar WhatsApp Cloud API en local, revisa la guía de túneles en [/whatsapp-tunnel](/whatsapp-tunnel).
- Si prefieres un flujo sin editar código, sigue la guía de [/getting-started-no-code](/getting-started-no-code).
