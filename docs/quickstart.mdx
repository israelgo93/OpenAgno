---
title: Inicio rapido
description: Instalar OpenAgno, crear un workspace usable y validar el primer arranque con la CLI actual.
---

## Requisitos

- Python 3.11 o superior
- Node.js 18+ solo si usas el bridge WhatsApp QR
- PostgreSQL con extension **pgvector** para knowledge vectorial
- SQLite solo si aceptas correr sin knowledge vectorial

## 1. Instala el proyecto

```bash
git clone https://github.com/israelgo93/OpenAgno.git
cd OpenAgno
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip install -e .
```

`bash setup.sh` sigue siendo valido si prefieres el instalador del repo.

## 2. Revisa los templates disponibles

```bash
openagno templates list
```

## 3. Inicializa un workspace

La ruta recomendada para empezar es copiar un template:

```bash
openagno init --template personal_assistant
```

Tambien puedes usar el onboarding interactivo:

```bash
openagno init
```

El onboarding historico via `python -m management.cli` sigue disponible, pero la ruta principal documentada desde esta version es `openagno`.

## 4. Valida antes de arrancar

```bash
openagno validate
```

## 5. Inicia el runtime

En foreground:

```bash
openagno start --foreground
```

Con supervisor en background:

```bash
openagno start
openagno status
openagno logs --follow
```

## 6. Verifica el primer arranque

```bash
curl http://127.0.0.1:8000/admin/health
```

Si el gateway esta sano, la respuesta incluye agentes, canales, modelo y scheduler.

## Variables de entorno

Copia `.env.example` a `.env` y completa las claves segun modelo y canales.

Variables comunes:

- `GOOGLE_API_KEY`, `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`
- `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`
- `WHATSAPP_ACCESS_TOKEN`, `WHATSAPP_PHONE_NUMBER_ID`, `WHATSAPP_VERIFY_TOKEN`, `WHATSAPP_APP_SECRET`
- `OPENAGNO_API_KEY` para proteger las rutas custom de knowledge
- `OPENAGNO_ROOT` si quieres fijar explicitamente la raiz del proyecto

## Recomendaciones para no perder tiempo

- Mantén el proyecto con la estructura estandar: `OPENAGNO_ROOT/workspace`.
- Usa templates para el primer arranque; evitas errores de estructura minima.
- Si el proyecto corre con SQLite, la knowledge vectorial se desactiva por diseño.
- Si usas WhatsApp Cloud API localmente, solo usa `WHATSAPP_SKIP_SIGNATURE_VALIDATION=true` en desarrollo.
