---
title: MCP / MCP e IDEs
description: Configure MCP in OpenAgno and connect the public docs to AI IDEs and assistants.
---

OpenAgno can use MCP in two layers:

1. **As a client**, by connecting to external servers declared in `workspace/mcp.yaml`.
2. **As a server**, by exposing the embedded runtime MCP endpoint when `agentos.enable_mcp_server` is enabled.

## English

### MCP client

Example:

```yaml
servers:
  - name: agno_docs
    transport: streamable-http
    url: https://docs.agno.com/mcp
    enabled: true
  - name: supabase
    transport: stdio
    command: npx
    args:
      - -y
      - "@supabase/mcp-server-supabase@latest"
      - --access-token
      - ${SUPABASE_ACCESS_TOKEN}
    enabled: false

expose:
  enabled: true
```

### What the loader supports

- `streamable-http`
- `sse`
- `stdio`
- `${VAR}` resolution inside strings, lists, and dictionaries
- per-server `env` for `stdio`
- declarative integration merging

### Embedded MCP server

When `agentos.enable_mcp_server` is enabled, AgentOS exposes its own MCP server next to the runtime routes.

That lets you:

- connect IDEs or external MCP clients
- use Studio with the local OS
- expose runtime tools and context in a controlled way

### OpenAgno Docs as an MCP server

The public docs are already available as a remote MCP server at:

```text
https://docs.openagno.com/mcp
```

It also exposes AI-friendly indexes at:

```text
https://docs.openagno.com/llms.txt
```

### Connect from Claude Code

```bash
claude mcp add openagno https://docs.openagno.com/mcp
```

### Connect from Cursor or VS Code

```bash
npx mint-mcp add openagno https://docs.openagno.com/mcp
```

### Recommendations

- Use `streamable-http` for remote servers when official support exists.
- Use `stdio` for local servers that depend on `npx` or local binaries.
- If an integration needs its own credentials, declare the MCP server under `workspace/integrations/<id>/integration.yaml`.
- For IDE-specific setup, also review [/ide-integration](/ide-integration).

## Español

OpenAgno puede usar MCP como cliente y como servidor. Como cliente, consume servidores externos declarados en `workspace/mcp.yaml`. Como servidor, expone el MCP embebido del runtime cuando `agentos.enable_mcp_server` está activo.

### Recomendaciones en español

- Usa `streamable-http` para servidores remotos.
- Usa `stdio` para servidores locales.
- Declara credenciales con `${VAR}` desde `.env`.
- Si vas a conectar IDEs, revisa también [/ide-integration](/ide-integration).
