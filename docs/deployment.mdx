## Modos soportados hoy

| Modo | Uso |
|------|-----|
| `openagno start --foreground` | desarrollo y debugging |
| `openagno start` | supervisor local |
| `systemd` | Linux con servicio persistente |
| `openagno deploy docker` | stack local con Docker Compose |

## Foreground

```bash
openagno start --foreground
```

Úsalo cuando quieras ver logs en tiempo real y depurar el runtime.

## Supervisor local

```bash
openagno start
openagno status
openagno logs --follow
openagno stop
```

El supervisor local funciona en Linux, macOS y Windows. En Windows usa el fallback nativo para arranque detached y parada por PID.

## systemd

```bash
sudo bash deploy/install-service.sh
sudo systemctl start openagno
sudo systemctl status openagno
journalctl -u openagno -f
```

`systemd` e `install-service.sh` siguen siendo **solo Linux**.

## Docker Compose

```bash
docker compose up -d db
openagno deploy docker
docker compose --profile qr up -d
```

## PyPI y builds

La publicación del paquete sigue en preparación. Hoy puedes validar el build localmente:

```bash
python -m build
```

Eso genera `sdist` y `wheel` para revisión del maintainer. La subida real a PyPI no forma parte del flujo público actual.

## Estado del despliegue AWS

La automatización AWS todavía no está implementada en la CLI. Sigue siendo parte del roadmap y no debe tratarse como capacidad cerrada.
